# 12 周学习路线

## 阶段目标

12 周后，完成一个可运行的 V1：

1. 支持录入热点
2. 支持生成选题和文案
3. 支持生成封面候选
4. 支持人工审核
5. 支持导出待发布内容

## 第 1 月：打基础和后端骨架

### 第 1 周

- 学 Python 工程化
- 学 FastAPI 路由、配置、日志
- 输出：能启动一个后端服务

### 第 2 周

- 学 PostgreSQL 与 SQLAlchemy
- 学 Redis 基础
- 输出：建好热点、选题、内容、审核四类核心表

### 第 3 周

- 学 HTTP 基础与 Playwright
- 输出：完成一个简单抓取脚本和登录态保存原型

### 第 4 周

- 学 LLM API 与 Prompt 设计
- 输出：输入热点文本，输出标题、正文、标签的 JSON 结果

## 第 2 月：跑通内容生产闭环

### 第 5 周

- 学选题拆解和风格模板设计
- 输出：一条热点生成多个内容角度

### 第 6 周

- 学封面模板与图片处理
- 输出：生成封面图和配图占位图

### 第 7 周

- 学 React/Next.js 基础
- 输出：完成审核台列表页和详情页

### 第 8 周

- 学 Celery 或轻量任务调度
- 输出：定时生成任务并处理失败重试

## 第 3 月：做可运营和可迭代能力

### 第 9 周

- 学内容风险控制
- 输出：敏感词检查、风险标签、人工复核拦截

### 第 10 周

- 学发布辅助工作流
- 输出：待发布队列和素材打包

### 第 11 周

- 学 SQL 分析和 pandas
- 输出：内容表现看板最小版

### 第 12 周

- 学 Docker 和部署
- 输出：V1 可部署版本

## 每周交付要求

每周至少产出一项可验证结果：

1. 一个可运行模块
2. 一份文档沉淀
3. 一个可演示界面
4. 一条真实数据流转链路

## 学习区检查清单

- 能解释清楚每个模块在整条链路中的位置
- 能独立启动后端和前端
- 能从热点文本生成结构化内容
- 能把内容存库并进入审核状态
- 能说明后续接入采集器和发布模块的边界

# 学习笔记

## 2026-04-29

### 今天新增代码对应的学习点

1. SQLAlchemy 2.x 基本分层

- `db/base.py` 负责声明 ORM 基类
- `db/session.py` 负责 Engine、Session 和建表初始化
- `models/` 负责数据库实体定义
- `schemas/` 负责接口输入输出的数据校验

2. FastAPI 路由拆分方式

- `api/router.py` 只负责集中注册
- `routes/*.py` 负责具体资源接口
- 当一个功能不再只是占位时，应该尽快从“内存列表”切换到真实数据库读写

3. 内容工作流的最小闭环

- 热点是起点
- 选题是热点的多个表达切角
- 文案是选题的多个表达版本
- 审核任务把热点、选题、文案串进人工决策环节

4. 前端单页工作台的理解方式

- 当前前端本质上是一个单页控制台
- `page.js` 同时承担状态、事件和接口调用
- 这种写法适合 V1 跑通流程，但后续需要逐步拆组件和 API 层

### 当前阶段最值得记住的点

1. 文档里写的“草案”不等于仓库现状，必须经常和 git 新增文件对齐
2. 这次新增文件说明项目已经从骨架阶段进入“最小闭环已落地”阶段
3. 后续学习重点应该从纯概念理解，转到真实 LLM 接入、异步任务和前端拆分

# 项目拆解与执行顺序

## 实施原则

1. 先做最短业务闭环
2. 先保证可控，再提升自动化
3. 每个阶段都要有可演示结果

## 阶段 1：V1 最小闭环

目标：完成从热点录入到审核出稿。

任务：

1. 搭建 FastAPI 后端骨架
2. 定义数据库模型和状态机
3. 完成热点录入 API
4. 完成选题生成接口占位
5. 完成文案生成接口占位
6. 完成审核台基础页面
7. 完成待审核列表和详情页

验收结果：

1. 可以创建热点
2. 可以生成模拟选题与文案结果
3. 可以进入待审核列表

## 阶段 2：内容生成增强

目标：接入真实模型能力。

任务：

1. 接入 LLM API
2. 增加 Prompt 模板管理
3. 增加结构化输出校验
4. 接入封面模板与图片生成

## 阶段 3：采集与工作流

目标：减少人工录入。

任务：

1. 接入热点采集器
2. 增加定时任务与重试
3. 增加素材存储与日志

## 阶段 4：发布辅助与数据闭环

目标：形成完整运营系统。

任务：

1. 增加发布辅助
2. 增加发布记录和数据回收
3. 增加策略优化面板

## 你当前应该先做什么

按优先级执行：

1. 启动后端骨架
2. 打通热点录入到审核任务创建
3. 再补文案和图片真实生成能力
4. 最后接采集和发布

# Frontend

V1 前端定位为审核台，目标不是大而全，而是先把内容确认流程做通。

## 当前骨架包含

1. Next.js App Router 应用入口
2. 单页三栏工作台首页
3. 全局样式与响应式布局
4. 热点录入表单
5. 热点列表与当前焦点区
6. 选题候选区
7. 文案候选区
8. 审核任务队列与通过/驳回操作

## 当前页面能力

前端当前已经可以直接调用后端接口，打通以下动作：

1. 创建热点
2. 为热点生成选题
3. 为选题生成文案
4. 为文案创建审核任务
5. 对审核任务执行通过或驳回
6. 显示热点数、待审核数、通过数、驳回数

## 运行方式

```bash
cd frontend
npm install
npm run dev
```

默认前端地址：

- `http://127.0.0.1:3000`

默认后端接口地址：

- `http://127.0.0.1:8000/api/v1`

如需切换后端地址，可以设置环境变量：

```bash
export NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000/api/v1
```

## 代码分布

- `app/page.js`：核心工作台逻辑、状态管理、事件处理、接口调用
- `app/globals.css`：页面视觉、三栏布局、卡片、按钮、表单样式
- `app/layout.js`：全局外壳和元信息
